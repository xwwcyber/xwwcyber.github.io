在日常的后端开发或数据处理工作中，熟练掌握 MySQL 的基础语法是必不可少的。本文为你整理了一份涵盖数据库、数据表、字段以及数据增删改查的基础命令速查表（Cheat Sheet），方便随时翻阅。

### 一、 登录与数据库操作 (Database Operations)

首先，我们需要进入 MySQL 环境并对数据库进行管理：

* **登录数据库**
    ```sql
    mysql -u root -p
    ```
* **查看所有数据库**
    ```sql
    SHOW DATABASES;
    ```
* **创建数据库**
    ```sql
    CREATE DATABASE db_name;
    ```
* **删除数据库**
    ```sql
    DROP DATABASE db_name;
    ```
* **查看数据库的创建信息**
    ```sql
    SHOW CREATE DATABASE db_name;
    ```
* **修改数据库字符集** (推荐使用 `utf8mb4`)
    ```sql
    ALTER DATABASE db_name DEFAULT CHARACTER SET utf8mb4;
    ```
* **切换/使用数据库**
    ```sql
    USE db_name;
    ```

  >注：如果你敲了一会儿代码，忘了自己现在到底在哪个数据库里，可以使用`SELECT DATABASE();`或者`status`来查询当前数据库

---

### 二、 数据表操作 (Table Operations)

进入具体的数据库后，就可以开始对表结构进行操作了：

* **查看当前数据库中的所有表**
    ```sql
    SHOW TABLES;
    ```
* **创建数据表** (需包含至少一个字段)
    ```sql
    CREATE TABLE table_name (
        id INT,
        name VARCHAR(50)
    );
    ```
* **查看表结构**
    ```sql
    DESC table_name;
    -- 或者使用详细结构查看：
    SHOW CREATE TABLE table_name;
    ```
* **删除数据表**
    ```sql
    DROP TABLE table_name;
    ```
* **复制表结构** (仅复制结构，不包含数据)
    ```sql
    CREATE TABLE new_table_name LIKE old_table_name;
    ```
* **复制表结构及数据**
    ```sql
    CREATE TABLE new_table_name AS SELECT * FROM old_table_name;
    ```

---

### 三、 字段操作 (Column Operations)

当表创建好后，如果需要调整表内的列（字段），可以使用 `ALTER TABLE` 命令：

* **添加新字段**
    ```sql
    ALTER TABLE table_name ADD column_name datatype;
    ```
* **修改字段数据类型或约束**
    ```sql
    ALTER TABLE table_name MODIFY column_name new_datatype;
    ```
* **修改字段名称及数据类型**
    ```sql
    ALTER TABLE table_name CHANGE old_column_name new_column_name datatype;
    ```
* **删除字段**
    ```sql
    ALTER TABLE table_name DROP COLUMN column_name;
    ```

---

### 四、 数据操作 - 增删改查 (Data Operations)

* **插入数据**
    ```sql
    INSERT INTO table_name (column1, column2) VALUES (value1, value2);
    ```
* **更新数据**
    ```sql
    UPDATE table_name SET column1 = new_value WHERE condition;
    ```
* **删除符合条件的数据** (只删数据，保留表结构)
    ```sql
    DELETE FROM table_name WHERE condition;
    ```
* **清空表中所有数据** (重置表，速度比 DELETE 快)
    ```sql
    TRUNCATE TABLE table_name;
    ```

---

### 五、 主键与约束 (Keys & Constraints)

合理的约束能保证数据的完整性：

* **添加单个主键**
    ```sql
    ALTER TABLE table_name ADD PRIMARY KEY (column_name);
    ```
* **添加联合主键** (两个字段组合在一起完全相同才算重复)
    ```sql
    ALTER TABLE table_name ADD PRIMARY KEY (column_name1, column_name2);
    ```
* **删除主键**
    ```sql
    ALTER TABLE table_name DROP PRIMARY KEY;
    ```
* **设置主键自动增长** (通常用于 INT 类型主键)
    ```sql
    ALTER TABLE table_name MODIFY column_name INT AUTO_INCREMENT;
    ```
* **设置字段默认值**
    ```sql
    ALTER TABLE table_name MODIFY column_name datatype DEFAULT default_value;
    ```

> **小贴士**：在执行 `UPDATE` 和 `DELETE` 操作时，千万别忘了加上 `WHERE` 条件，否则会修改或删除整张表的数据哦！

### 六、 数据查询 (Data Query Language - DQL)

查询是数据库操作的重中之重。下面整理了从基础到进阶的 `SELECT` 查询语法：

#### 1. 基础查询
* **查询表中的所有字段**
    ```sql
    SELECT * FROM table_name;
    ```
* **查询指定的字段**
    ```sql
    SELECT column1, column2 FROM table_name;
    ```
* **字段去重查询** (去除重复记录)
    ```sql
    SELECT DISTINCT column_name FROM table_name;
    ```

#### 2. 条件查询 (WHERE)
* **基础条件过滤** (`=`, `>`, `<`, `>=`, `<=`, `!=` 或 `<>`)
    ```sql
    SELECT * FROM table_name WHERE age >= 18;
    ```
* **模糊查询** (`LIKE`，`%` 代表任意多个字符，`_` 代表单个字符)
    ```sql
    SELECT * FROM table_name WHERE name LIKE '张%'; -- 查找姓张的人
    ```
* **范围查询** (`IN`, `BETWEEN AND`)
    ```sql
    SELECT * FROM table_name WHERE id IN (1, 3, 5);
    SELECT * FROM table_name WHERE age BETWEEN 20 AND 30;
    ```
* **逻辑组合** (`AND`, `OR`, `NOT`)
    ```sql
    SELECT * FROM table_name WHERE age > 18 AND gender = 'M';
    ```

#### 3. 排序查询 (ORDER BY)
* **单字段排序** (`ASC` 为升序，`DESC` 为降序。默认是升序)
    ```sql
    SELECT * FROM table_name ORDER BY age DESC;
    ```
* **多字段排序** (先按第一个字段排，相同的再按第二个字段排)
    ```sql
    SELECT * FROM table_name ORDER BY age DESC, id ASC;
    ```

#### 4. 聚合函数
用于对数据进行统计，聚合函数会自动忽略 `NULL` 值。
* `COUNT()`: 统计数量
* `MAX()` / `MIN()`: 最大/最小值
* `SUM()`: 求和
* `AVG()`: 平均值
    ```sql
    SELECT COUNT(*) FROM table_name; -- 统计总行数
    SELECT MAX(score) FROM table_name WHERE class_id = 1;
    ```

#### 5. 分组查询 (GROUP BY)
通常配合聚合函数使用，用于按某个字段对数据进行分类统计。
* **基础分组**
    ```sql
    SELECT gender, COUNT(*) FROM table_name GROUP BY gender;
    ```
* **分组后过滤 (HAVING)** *(注意：`WHERE` 用于分组前的条件过滤，`HAVING` 用于分组后的结果过滤)*
    ```sql
    -- 统计每个班级的人数，并只显示人数大于 40 人的班级
    SELECT class_id, COUNT(*) AS count FROM table_name GROUP BY class_id HAVING count > 40;
    ```

#### 6. 分页查询 (LIMIT)
在前端页面做数据分页时必不可少的语法。
* **语法**: `LIMIT 偏移量, 查询记录数`
    ```sql
    SELECT * FROM table_name LIMIT 0, 10; -- 查询第 1 到 10 条数据
    SELECT * FROM table_name LIMIT 10, 10; -- 查询第 11 到 20 条数据
    ```

#### 7. 多表连接查询 (JOIN)

在实际业务中，数据往往分散在多张表中，我们需要通过 JOIN 来联合查询：
* **内连接 (INNER JOIN)**: 只返回两张表中互相匹配的数据。
    ```sql
    SELECT t1.name, t2.score 
    FROM student t1 
    INNER JOIN exam t2 ON t1.id = t2.student_id;
    ```
* **左外连接 (LEFT JOIN)**: 返回左表的所有数据。如果右表中没有匹配项，则显示为 NULL。
    ```sql
    SELECT t1.name, t2.score 
    FROM student t1 
    LEFT JOIN exam t2 ON t1.id = t2.student_id;
    ```
