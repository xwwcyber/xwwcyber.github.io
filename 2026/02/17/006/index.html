一次 GitHub 项目实战全流程

编辑原文
2026-02-17 tips 683 字
PR

1.需要先把这个项目fork一下

2.需要把这个项目下载到本地，git clone一下，确保项目能够正常运行

git clone https://github.com/xwwcyber/tinyserial.git

3.打开这个项目的issue（问题/议题），点击open

Open 和 Closed 代表了这些任务的当前状态。
Open (开启中)：这代表尚未解决或正在进行中的事项
Closed (已关闭)：这代表已经处理完毕或已终止的事项

4.找到你想解决或者你可以解决的问题

确定好问题之后你就可以着手解决你选择的问题了，你可以使用claude来看一下代码的整体框架，然后解决这个问题

5.之后就是把代码提交上去

打开你在本地运行成功的项目，右键、打开git bash here

查询当前远程连接库状态git remote -v

如果你看到的是 https://github.com/carloscn/tinyserial.git,
说明你现在连的是“原作者”的仓库（你没有权限推送到那里）。

你需要执行下面这行命令（把地址换成你 Fork 后的地址）：

git remote set-url origin https://github.com/xwwcyber/tinyserial.git
如果你看到的是 https://github.com/xwwcyber/tinyserial.git,
说明已经连上你自己的仓库了，直接去下一步。

上传代码 (三连击)：

依次输入以下三行命令（每输一行按一次回车）

git add .
git commit -m "Fix: Hex mode bug and add upper case display"
git push origin master

6.需要把代码上传到原项目

第一步：打开你的 GitHub 仓库页面

在浏览器地址栏输入：（你fork的仓库地址）
👉 https://github.com/xwwcyber/tinyserial

第二步：点击那个“显眼”的按钮

当你进入页面后，GitHub 会自动检测到你刚才上传了新代码。你会看到页面上方出现了一个黄色的提示条，上面有一个绿色的按钮写着 “Compare & pull request”，直接点击这个绿色按钮。

(如果没有看到这个黄色条提示条，请点击页面上的 “Contribute” 标签 -> 然后点击绿色的 “Open pull request” 按钮)

第三步：填写“交卷说明” (PR 描述)

你会进入一个编辑页面。这里是写给原作者看的，告诉他你干了什么好事。

Title (标题):概括一下你做了什么

Leave a comment (描述):请使用英文描述（显得很专业）

第四步：点击 “Create pull request”

检查一下页面下方显示的代码变动（Diff），是不是你修改的那几行（红色是删除的旧代码，绿色是新增的新代码）。

确认无误后，点击页面右下角的绿色大按钮 Create pull request。
