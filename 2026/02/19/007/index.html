忘记 MySQL 8.0 root 密码？教你如何重置密码

编辑原文
2026-02-19 tips 519 字
MySQL

1.我虚拟机的环境

OS: Ubuntu Linux

Database: MySQL 8.0+

2.故障现象

尝试登录 MySQL 时报错

ERROR 1045 (28000): Access denied for user 'root'@'localhost' (using password: NO)

显然是密码错了，或者没权限。

3.解决步骤

第一步：彻底停止 MySQL 服务

首先需要停止正常运行的服务。为了防止有残留进程导致后续冲突（提示 A mysqld process already exists），建议直接查杀

sudo pkill -9 mysqld
sudo systemctl stop mysql

第二步：解决 Socket 目录丢失问题

在尝试启动免密模式（Safe Mode）时，我遇到了这个报错

mysqld_safe Directory '/var/run/mysqld' for UNIX socket file don't exists.

原因：MySQL 服务停止后，Linux 系统有时会自动清理 /var/run/ 下的临时目录。

解决：必须手动创建该目录并赋予权限，否则服务起不来。

执行下面两行命令

sudo mkdir -p /var/run/mysqld
sudo chown mysql:mysql /var/run/mysqld

第三步：启动“跳过授权表”模式

执行以下命令，让 MySQL 在启动时不检查密码

sudo mysqld_safe --skip-grant-tables &

注：末尾的 & 是为了让进程在后台运行。

此时可以直接免密登录

mysql -u root

第四步：重置密码

  1. 先刷新权限（必须做！）
FLUSH PRIVILEGES;

2. 确认用户 Host（非常重要！）

先看一眼你的 root 用户到底是属于 localhost 还是 %（任意主机）

USE mysql;
SELECT user, host FROM user;

我的情况是：root 对应的 host 是 %，而不是默认的 localhost。如果这时候强行改 localhost 的密码，就会报找不到用户。

3. 执行修改命令,根据上一步查询到的 host（我的是 %），执行

ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '你的新密码';

注意：如果你的 host 是 localhost，就将 % 换成 localhost。

  1. 再次刷新并退出
FLUSH PRIVILEGES;
exit;

第五步：重启并验证

密码修改成功后，需要关掉免密模式，恢复正常服务。

# 杀掉后台进程
sudo pkill -9 mysqld

# 正常启动服务
sudo systemctl start mysql

# 使用新密码登录
mysql -u root -p
