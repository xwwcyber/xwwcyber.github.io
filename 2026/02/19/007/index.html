忘记 MySQL 8.0 root 密码？教你如何重置密码

编辑原文
2026-02-19 tips 803 字
MySQL

1.环境说明

  • OS: Ubuntu Linux

  • Database: MySQL 8.0+

2.故障现象

尝试登录 MySQL 时报错

ERROR 1045 (28000): Access denied for user 'root'@'localhost' (using password: NO)

显然是密码错误或权限问题。

3.完整解决步骤

第一步：彻底停止 MySQL 服务

为了确保进入安全模式时不发生冲突（避免提示 A mysqld process already exists），建议彻底查杀并停止服务

# 强制杀掉所有 mysqld 进程
sudo pkill -9 mysqld
# 确保服务停止
sudo systemctl stop mysql

第二步：解决 Socket 目录丢失问题

在尝试启动免密模式（Safe Mode）时，我遇到了这个报错

mysqld_safe Directory '/var/run/mysqld' for UNIX socket file don't exists.

原因：MySQL 服务停止后，Linux 系统有时会自动清理/var/run/下的临时目录。

解决：必须手动创建该目录并赋予mysql权限，否则服务起不来。

执行下面两行命令修复

sudo mkdir -p /var/run/mysqld
sudo chown mysql:mysql /var/run/mysqld

第三步：启动“跳过授权表”模式

执行以下命令，让 MySQL 在启动时跳过密码验证

sudo mysqld_safe --skip-grant-tables &

注：末尾的 & 是为了让进程在后台运行。

此时，你可以直接免密登录数据库

mysql -u root

第四步：重置密码

进入 MySQL 命令行后，请严格按照以下顺序操作：

  1. 先刷新权限（必须做！）
FLUSH PRIVILEGES;

2. 确认用户 Host（非常重要！）

先看一眼你的 root 用户到底是属于 localhost 还是%（任意主机）

很多教程直接让你改 localhost 的密码，但如果你的 root 用户允许远程登录（Host 为 %），命令就会报错。先查看当前用户配置：

USE mysql;
SELECT user, host FROM user;

我的情况是：root 对应的 host 是 %，而不是默认的 localhost。

如果这时候强行改'root'@'localhost' 的密码，就会报找不到用户的错误（如下图）。



3. 执行修改命令

根据上一步查询到的 host（我的是 %），执行密码更新语句：

ALTER USER 'root'@'%' IDENTIFIED WITH mysql_native_password BY '你的新密码';

注意：如果你的 host 是 localhost，就将 % 换成 localhost。

  1. 再次刷新并退出
FLUSH PRIVILEGES;
exit;

🆘 救急 Plan B：如果执行 ALTER USER 报错 1396

如果你确认 Host 写对了，但依然报错 ERROR 1396，可以尝试先清空密码，正常登录后再修改：

  1. 执行 UPDATE user SET authentication_string='' WHERE user='root';

  2. 退出并重启 MySQL服务（正常启动，不带免密参数）。

  3. 输入 mysql -u root （此时不需要密码即可直接回车登录）。

  4. 登录后使用 ALTER USER 'root'@'%' IDENTIFIED BY '新密码'; 设置密码。

第五步：重启并验证

密码修改成功后，需要关掉免密模式，恢复正常服务。

# 杀掉后台进程
sudo pkill -9 mysqld

# 正常启动服务
sudo systemctl start mysql

# 使用新密码登录
mysql -u root -p

4.总结

这次重置密码主要踩了两个坑：

  • Socket 目录消失：需要手动重建 /var/run/mysqld。
  • Host 匹配问题：一定要先 SELECT user, host FROM user 确认 root 账号的 Host 归属，不要盲目照搬网上的'root'@'localhost'命令。
