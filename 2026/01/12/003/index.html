贪吃蛇小游戏开发总结：从零到上线的技术实践

项目简介

这是一个基于 uni-app 框架开发的贪吃蛇小游戏，采用 Vue3 + Canvas 2D 技术栈，支持虚拟摇杆控制、穿墙功能、特殊食物系统等特色功能。项目已成功部署到 GitHub：xwwcyber/snake_game

开发过程中遇到的主要问题及解决方案

1. 摇杆控制系统的实现

问题描述

最初使用传统的方向按键控制，用户体验不够流畅，需要改为虚拟摇杆控制，并且要求用户松开摇杆时蛇保持当前方向继续移动。

技术难点

  • 触摸事件的精确捕获和处理
  • 摇杆位置的计算和限制
  • 方向判断的准确性

解决方案

实现触摸事件监听：

onJoystickStart(e) {
  if (!this.gameRunning) return;
  this.isJoystickActive = true;
  if (!this.joystickBaseRect) {
    this.cacheJoystickPosition();
  }
}

onJoystickMove(e) {
  if (!this.gameRunning || !this.isJoystickActive || !this.joystickBaseRect) return;
  
  const touch = e.touches[0];
  const centerX = this.joystickBaseRect.centerX;
  const centerY = this.joystickBaseRect.centerY;
  
  let deltaX = touch.clientX - centerX;
  let deltaY = touch.clientY - centerY;
  
  // 限制最大移动距离
  const distance = Math.sqrt(deltaX * deltaX + deltaY * deltaY);
  const maxDistancePx = this.joystickMaxDistance / 2;
  
  if (distance > maxDistancePx) {
    deltaX = (deltaX / distance) * maxDistancePx;
    deltaY = (deltaY / distance) * maxDistancePx;
  }
  
  this.joystickX = deltaX * 2;
  this.joystickY = deltaY * 2;
  
  this.updateSnakeDirection(deltaX, deltaY);
}

onJoystickEnd() {
  this.isJoystickActive = false;
  this.joystickX = 0;
  this.joystickY = 0;
  // 关键：不重置蛇的方向，保持当前方向
}

方向判断优化：

updateSnakeDirection(deltaX, deltaY) {
  const threshold = 5; // 阈值，避免微小移动导致方向变化
  
  if (Math.abs(deltaX) < threshold && Math.abs(deltaY) < threshold) {
    return;
  }
  
  // 判断主要方向
  if (Math.abs(deltaX) > Math.abs(deltaY)) {
    if (deltaX > 0 && this.dx !== -1) {
      this.dx = 1;
      this.dy = 0;
    } else if (deltaX < 0 && this.dx !== 1) {
      this.dx = -1;
      this.dy = 0;
    }
  } else {
    if (deltaY > 0 && this.dy !== -1) {
      this.dx = 0;
      this.dy = 1;
    } else if (deltaY < 0 && this.dy !== 1) {
      this.dx = 0;
      this.dy = -1;
    }
  }
}

关键点：

  • 使用阈值避免微小移动导致的误操作
  • 防止蛇直接反向移动（如向右时不能直接向左）
  • 松开摇杆时不重置方向，保持当前移动方向

2. Canvas 绘制显示问题

问题描述

游戏运行时，蛇的身体和食物在 Canvas 上不可见，只能看到背景色。

原因分析

  1. Canvas 初始化时机不当，在 DOM 未完全渲染时就尝试绘制
  2. 像素比例（rpx 到 px）转换不正确
  3. 绘制后没有调用 ctx.draw() 方法

解决方案

调整 Canvas 初始化时机：

onReady() {
  // 在 onReady 中初始化，确保 DOM 已经渲染完成
  this.initGame();
  this.loadHighScore();
}

initGame() {
  this.ctx = uni.createCanvasContext('gameCanvas', this);
  
  // 缓存 tile 大小计算
  this.tileSize = this.canvasSizePx / this.tileCount;
  
  this.generateFood();
  this.cacheJoystickPosition();
  
  // 延迟绘制，确保 canvas 初始化完成
  setTimeout(() => {
    this.drawGame();
  }, 100);
}

正确处理像素比例：

data() {
  return {
    canvasSize: 400,    // rpx 单位（用于样式）
    canvasSizePx: 200, // px 单位（用于绘制，400rpx ≈ 200px）
    tileCount: 20,
    tileSize: 0        // 缓存计算值
  }
}

drawGame() {
  // 清空画布 - 使用 px 单位
  this.ctx.fillStyle = '#2c3e50';
  this.ctx.fillRect(0, 0, this.canvasSizePx, this.canvasSizePx);
  
  // 绘制蛇
  const size = this.tileSize - 1;
  this.snake.forEach((segment, index) => {
    const x = segment.x * this.tileSize;
    const y = segment.y * this.tileSize;
    this.ctx.fillRect(x, y, size, size);
  });
  
  // 重要：调用 draw() 方法将绘制内容显示到 canvas
  this.ctx.draw();
}

关键点：

  • 使用 onReady 生命周期钩子确保 DOM 渲染完成
  • 区分 rpx（样式）和 px（绘制）单位
  • 缓存计算结果避免重复计算
  • 必须调用 ctx.draw() 才能显示绘制内容

3. 食物生成逻辑问题

问题描述

蛇吃掉一个食物后，没有新的食物出现，导致游戏无法继续。

原因分析

  1. 食物生成逻辑过于复杂，包含碰撞检测
  2. 生成失败时没有重试机制
  3. 代码逻辑与注释不一致

解决方案

简化食物生成逻辑：

generateFood() {
  // 简单随机生成，允许出现在蛇身上
  this.foodX = Math.floor(Math.random() * this.tileCount);
  this.foodY = Math.floor(Math.random() * this.tileCount);
  
  // 边界检查，确保食物在有效范围内
  this.foodX = Math.max(0, Math.min(this.foodX, this.tileCount - 1));
  this.foodY = Math.max(0, Math.min(this.foodY, this.tileCount - 1));
  
  console.log('新食物生成成功:', { x: this.foodX, y: this.foodY });
}

确保食物生成调用：

moveSnake() {
  // ... 其他逻辑
  
  // 检查吃到食物
  if (head.x === this.foodX && head.y === this.foodY) {
    this.score += 10;
    console.log('吃到食物！当前分数:', this.score);
    // 重要：生成新食物
    this.generateFood();
  } else {
    this.snake.pop();
  }
  
  this.drawGame();
}

关键点：

  • 简化生成逻辑，移除不必要的碰撞检测
  • 添加边界检查确保食物在有效范围内
  • 添加日志便于调试
  • 确保每次吃掉食物后都调用生成方法

4. 允许食物出现在蛇身上

问题描述

为了增加游戏挑战性，需要允许食物出现在蛇的身体上，并用不同颜色区分。

技术难点

  • 食物与蛇身重叠时的视觉区分
  • 保持游戏逻辑的一致性

解决方案

食物颜色区分：

drawGame() {
  // ... 绘制蛇
  
  // 绘制食物 - 允许出现在蛇身上，使用特殊颜色区分
  let foodColor = '#e74c3c'; // 默认红色
  
  // 检查食物是否在蛇身上
  const onSnakeBody = this.snake.some(segment => 
    segment.x === this.foodX && segment.y === this.foodY
  );
  
  if (onSnakeBody) {
    foodColor = '#f39c12'; // 橙色，表示在蛇身上的食物
  }
  
  this.ctx.fillStyle = foodColor;
  this.ctx.fillRect(
    this.foodX * this.tileSize,
    this.foodY * this.tileSize,
    size,
    size
  );
  
  this.ctx.draw();
}

UI 提示：

<view class="instructions">
  <text class="instruction-text">使用摇杆控制蛇的移动方向</text>
  <text class="instruction-text">食物可以出现在蛇身上，增加挑战性！</text>
</view>

关键点：

  • 使用 Array.some() 方法高效检测重叠
  • 用不同颜色（红色 vs 橙色）区分食物状态
  • 在 UI 中添加提示说明
  • 保持游戏逻辑简单，不增加复杂度

5. 穿墙功能的实现

问题描述

传统的贪吃蛇碰到墙壁就会死亡，为了增加游戏趣味性，实现穿墙功能（从一边边界出去，从对面边界出现）。

解决方案

moveSnake() {
  if (this.dx === 0 && this.dy === 0) return;
  
  const head = { x: this.snake[0].x + this.dx, y: this.snake[0].y + this.dy };
  
  // 穿墙处理 - 从对面边界出现
  if (head.x < 0) {
    head.x = this.tileCount - 1;
  } else if (head.x >= this.tileCount) {
    head.x = 0;
  }
  
  if (head.y < 0) {
    head.y = this.tileCount - 1;
  } else if (head.y >= this.tileCount) {
    head.y = 0;
  }
  
  // 检查碰撞自己
  for (let segment of this.snake) {
    if (head.x === segment.x && head.y === segment.y) {
      this.endGame();
      return;
    }
  }
  
  // ... 其他逻辑
}

关键点：

  • 在碰撞检测之前处理穿墙逻辑
  • 使用边界值 0 和 tileCount - 1
  • 穿墙不影响自身碰撞检测
  • 提升游戏可玩性和趣味性

6. 性能优化

问题描述

摇杆控制时频繁进行 DOM 查询，导致性能下降。

解决方案

缓存摇杆基座位置：

data() {
  return {
    joystickBaseRect: null // 缓存摇杆基座位置
  }
},

initGame() {
  // ... 其他初始化
  this.cacheJoystickPosition();
},

cacheJoystickPosition() {
  const query = uni.createSelectorQuery().in(this);
  query.select('.joystick-base').boundingClientRect(rect => {
    if (rect) {
      this.joystickBaseRect = {
        centerX: rect.left + rect.width / 2,
        centerY: rect.top + rect.height / 2
      };
    }
  }).exec();
},

onJoystickMove(e) {
  // 使用缓存的摇杆基座位置，避免频繁 DOM 查询
  if (!this.joystickBaseRect) return;
  
  const centerX = this.joystickBaseRect.centerX;
  const centerY = this.joystickBaseRect.centerY;
  // ... 其他逻辑
}

缓存计算值：

data() {
  return {
    tileSize: 0 // 缓存每个格子的像素大小
  }
},

initGame() {
  // 缓存 tile 大小计算
  this.tileSize = this.canvasSizePx / this.tileCount;
  // ... 其他初始化
},

drawGame() {
  // 使用缓存的值，避免重复计算
  const size = this.tileSize - 1;
  // ... 绘制逻辑
}

关键点：

  • 缓存 DOM 查询结果，避免重复查询
  • 缓存计算结果，减少重复计算
  • 在初始化时一次性完成缓存
  • 显著提升性能，减少卡顿

7. 代码质量改进

问题描述

代码检查时发现一些潜在问题，需要修复以提升代码质量。

解决方案

添加错误处理：

drawGame() {
  if (!this.ctx) return;
  
  try {
    // 绘制逻辑
    this.ctx.fillStyle = '#2c3e50';
    this.ctx.fillRect(0, 0, this.canvasSizePx, this.canvasSizePx);
    // ... 其他绘制
    this.ctx.draw();
  } catch (error) {
    console.error('绘制游戏失败:', error);
  }
}

添加边界检查：

generateFood() {
  this.foodX = Math.floor(Math.random() * this.tileCount);
  this.foodY = Math.floor(Math.random() * this.tileCount);
  
  // 边界检查，确保食物在有效范围内
  this.foodX = Math.max(0, Math.min(this.foodX, this.tileCount - 1));
  this.foodY = Math.max(0, Math.min(this.foodY, this.tileCount - 1));
}

完善日志系统：

console.log('游戏绘制完成', {
  snakeLength: this.snake.length,
  foodPosition: { x: this.foodX, y: this.foodY },
  foodOnSnake: onSnakeBody,
  canvasSizePx: this.canvasSizePx,
  tileCount: this.tileCount
});

关键点：

  • 使用 try-catch 保护关键操作
  • 添加边界检查防止越界
  • 完善日志系统便于调试
  • 提升代码健壮性和可维护性

技术栈总结

  • 框架: uni-app (Vue3)
  • 渲染: Canvas 2D API
  • 存储: uni-app 本地存储 API
  • 样式: rpx 响应式单位
  • 版本控制: Git + GitHub

项目亮点

  1. 摇杆控制系统: 流畅的虚拟摇杆，支持松开保持方向
  2. 穿墙功能: 增加游戏趣味性和可玩性
  3. 特殊食物系统: 食物可出现在蛇身上，用颜色区分
  4. 性能优化: 缓存机制，减少重复计算和 DOM 查询
  5. 完善的错误处理: try-catch 保护，边界检查
  6. 详细的日志系统: 便于调试和问题排查

经验总结

  1. Canvas 渲染时序很重要: 必须在 DOM 完全渲染后才能初始化 Canvas
  2. 单位转换要明确: 区分 rpx（样式）和 px（绘制）单位
  3. 性能优化从细节做起: 缓存机制可以显著提升性能
  4. 用户体验至上: 摇杆控制的阈值设置、方向保持等细节都很重要
  5. 代码质量不容忽视: 错误处理、边界检查、日志系统都很重要
  6. 持续迭代改进: 从基础功能到特色功能，逐步完善

后续优化方向

  • 添加音效系统
  • 实现难度选择功能
  • 添加排行榜功能
  • 优化游戏动画效果
  • 支持多平台发布（H5、小程序等）
  • 添加游戏教程和帮助文档

项目地址

  • GitHub: https://github.com/xwwcyber/snake_game

结语

这个小游戏是我完全使用AI完成的，使用claude实现的贪吃蛇功能。
